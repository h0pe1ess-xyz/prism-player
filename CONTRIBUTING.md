# Contributing to Prism Player

First of all, thank you for considering contributing to Prism Player! It's people like you that make open-source software such a great community to learn, inspire, and create.

## 🧠 Code of Conduct
By participating in this project, you are expected to uphold our Code of Conduct:
- Be respectful and welcoming to others.
- Accept constructive criticism gracefully.
- Focus on what is best for the community.

## 🐞 How to Contribute

### 1. Reporting Bugs
Bugs are tracked as GitHub issues. When creating an issue, please:
- Use the **Bug Report** template.
- Provide clear steps to reproduce the issue.
- Include your OS, terminal emulator, and Python version.
- Include crash logs if applicable.

### 2. Suggesting Enhancements
Enhancement suggestions are tracked as GitHub issues.
- Use the **Feature Request** template.
- Explain clearly *why* this feature would be useful to most users.
- Provide mockups or ASCII concepts if it involves UI changes.

### 3. Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation.
4. Ensure your code follows the existing style guidelines (PEP 8 for Python).
5. Open a Pull Request using the PR template.

## 🏗️ Architecture Overview
Please read [`docs/ARCHITECTURE.md`](docs/ARCHITECTURE.md) to understand how the codebase is structured before submitting large changes.
